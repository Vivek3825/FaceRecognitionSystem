4. DETAILED SYSTEM COMPONENTS
================================================================================

4.1 FACE DETECTION MODULE (detect_faces.py)
-------------------------------------------
PURPOSE: Extract and crop faces from raw images using MTCNN

KEY FEATURES:
- Automatic face detection in images with multiple faces or backgrounds
- Face cropping and standardization to 160x160 pixels
- Comprehensive error handling for missing/corrupted images
- Batch processing with progress feedback
- Consistent naming convention: P{ID}_{pose}.jpg

TECHNICAL IMPLEMENTATION:
- MTCNN Configuration:
  * keep_all=False (single best face per image)
  * device='cpu' (CPU processing for stability)
  * post_process=True (automatic normalization)
  * image_size=160 (standardized output size)

INPUT: Raw images from dataset/images/ (42 images, 14 people, 3 poses each)
OUTPUT: Cropped face images in dataset/faces/ + updated face_info.csv

PROCESSING PIPELINE:
1. Read original info.csv with raw image paths
2. Load each image and detect faces using MTCNN
3. Extract and normalize the best face detection
4. Save cropped face with standardized naming
5. Create face_info.csv mapping to processed faces

4.2 FEATURE EXTRACTION MODULE (extract_features.py)
---------------------------------------------------
PURPOSE: Generate 128-dimensional embeddings using pre-trained FaceNet

KEY FEATURES:
- InceptionResnetV1 model pre-trained on VGGFace2
- Batch processing with tqdm progress tracking
- Proper tensor normalization and preprocessing
- Efficient storage in compressed NPZ format
- Metadata management with CSV mapping

TECHNICAL IMPLEMENTATION:
- Model: InceptionResnetV1(pretrained='vggface2').eval()
- Input Processing:
  * RGB image conversion
  * Tensor permutation (HWC → CHW)
  * Normalization: (pixel/255.0 - 0.5) / 0.5
  * Resize to 160x160 if needed
- Dynamic device detection (CPU/GPU)

INPUT: Cropped faces from face_info.csv
OUTPUT: 
- all_embeddings.npz: {key: 128d_vector} mapping
- embeddings.csv: metadata with embedding keys

EMBEDDING PROCESS:
1. Read face_info.csv for processed face paths
2. Load and preprocess each face image
3. Extract 128-dimensional embedding using FaceNet
4. Store with base filename as key (e.g., "P001_p1_front")
5. Save both numerical data (NPZ) and metadata (CSV)

4.3 FACE MATCHING MODULE (match_face.py)
----------------------------------------
PURPOSE: Compare test faces against known database using similarity metrics

KEY FEATURES:
- Cosine similarity comparison algorithm
- Configurable similarity threshold (0.6)
- Automatic test face detection and preprocessing
- Metadata retrieval for matched identities
- Unknown face detection for low-confidence matches

MATCHING ALGORITHM:
1. Load all stored embeddings from NPZ file
2. Load metadata mapping from CSV file
3. Detect and extract face from test image using MTCNN
4. Generate embedding for test face using FaceNet
5. Compare against all known embeddings using cosine similarity
6. Return best match if above threshold, otherwise "Unknown"

SIMILARITY CALCULATION:
similarity = F.cosine_similarity(test_embedding, known_embedding, dim=0)

THRESHOLD LOGIC:
- similarity >= 0.6: Valid match
- similarity < 0.6: Unknown face

INPUT: Test image path
OUTPUT: Matched person name, ID, confidence score, or "Unknown"

4.4 REAL-TIME RECOGNITION MODULE (real_time_recognition.py)
----------------------------------------------------------
PURPOSE: Live face recognition through webcam with multiple face support

KEY FEATURES:
- Real-time webcam face detection and tracking
- Multiple simultaneous face recognition
- Confidence-based visual feedback with color coding
- Performance optimization for smooth real-time operation
- Margin-based matching for reduced false positives

ADVANCED MATCHING LOGIC:
- High Confidence (Green): score ≥ 0.5 AND margin ≥ 0.05
- Medium Confidence (Yellow): score ≥ 0.4 AND margin ≥ 0.08
- Low Confidence (Red): Below thresholds → "Unknown"

PERFORMANCE OPTIMIZATIONS:
1. Single normalization pipeline (removed double normalization)
2. Lower detection threshold (0.85) for better face detection
3. GPU acceleration when available
4. Efficient embedding comparison using vectorized operations
5. Minimum face size filtering (80x80 pixels)

REAL-TIME PROCESSING PIPELINE:
1. Capture frame from webcam
2. Detect all faces using MTCNN
3. Filter faces by confidence and size
4. Extract embeddings for each valid face
5. Compare against known database
6. Display results with bounding boxes and labels
7. Apply confidence-based color coding

VISUAL FEEDBACK:
- Green Box: High confidence match
- Yellow Box: Medium confidence match  
- Red Box: Low confidence / Unknown
- Text Display: Name and confidence score
